  276～352 : 地図作成に用いる点群が増えすぎないように，フィルタリング(計算量の削減)
  361～412 : 地図作成を行う．グリッドで分割された平面空間があり，それぞれのセルにはそのセルからz方向にある点群が，z座標の大きい順に格納されている．隣り合う点同士の差を求め，その差の中で最も大きいものを，そのセルにおける隙間として，隙間を構成する2点のz座標(高い方をz1，低い方をz2とする)を保持する．センサーの取得状況によっては，ドローン位置から高い位置もしくは低い位置に点群が取得されない場合がある．その場合は，ドローンの位置(z座標)からz方向に安全マージン分伸ばした値を，隙間を構成する2点のうちの1点，もしくは2点のz座標として保持する．
  415～454 : 地図作成の続き．1つのセルで隙間を見つけても，セルの大きさはドローンの機体幅よりも小さいのでドローンが飛行する通路を発見することはできない．ここでは，セルごとに保持した隙間をつなげて，ドローンが飛行する通路を発見する．具体的には，あるセルを中心としてドローンの機体幅(円と考える)とかぶるセルすべてにおいて，それぞれに保持されたz1の中で最も低い点を天井とし，z2の中で最も高い点を床として保持する．この天井と床のz方向の差がドローンの機体の高さより大きければ，そのセルの上はドローンが飛行可能と判定することができる．これをすべてのセルで計算することで，飛行可能と判定されたセルによる通路がわかる．
  456～498 : 飛行可能と判定されたセルを青，飛行不可と判定されたセルを赤として可視化する．
  502～645 : ゴールを受け取り，経路計画(astar_planner)し，現在地，ゴール地点，経路を可視化する．


# astar_planner.cppにおいて書いたコード
 →65～123 : 経路計画を行う．通路を発見することはできたが，通路から次の通路へ移動するには，通路同士が繋がっていなければいけない．ここでは，あるセルから8方向の隣り合うセルにおいて，低い方の天井をその2つのセルにおける天井とし，高い方の床をその2つのセルにおける床とする．この新たな天井と床のz方向の差がドローンの機体の高さより大きければ，その2つのセル間の移動が可能であると判定できる．これをすべての隣り合うセル同士で行い，ドローンが移動可能な真の通路がわかる．あとはそれぞれのセルをノードとしてA*による経路計画を行う．このとき，未知領域(点群が全く取得できていない領域)に経路が生成されないように，Receding Horizon戦略をとる．具体的には，あるセルから8方向探索時に，いずれかの方向にセルがなければそのセルが既知領域における先端部(既知領域と未知領域の境界)にあると判定する．経路計画はこの先端部まで行う．これにより，認識できる安全な空間のみで経路を生成し，ドローンを安全に飛行させることが可能となる．
